  Meta Dataset
A  Meta Dataset is a Physical Data Entity that represents the structure of a collection of related data held in a data source. The represented data structure can be relational (Table) or NoSQL (triple store, document store, flat files, etc.).
A  Meta Dataset consists of three types of MetaData Field:
1) An Identification MetaField, sometimes called "primary key".
2) Relationship MetaFields used to represent relationships with other  Meta Datasets.
3) Value MetaFields used to store values in the  Meta Dataset.
 Meta Datasets can be defined at design time in the context of data architecture activities. They can also be imported automatically through data discovery.

  Conceptual Object
A  Conceptual Object is the representation of any tangible or intanglible resource, or its respective state, that is critical for comprehending an enterprise, including its data, resources, and activities.
Similar to any Information Object, a  Conceptual Object can be classified into three categories:
1)  Conceptual Entitys denote entities that can change over time.
2) Event Concepts embody the temporal boundaries associated with Conceptual Entitys.
3) Concept Propertys represent immutable characteristics of Conceptual Entitys.

  Data Assurance Case A  Data Assurance Case is a structured argument, supported by evidence, intended to justify that a data is acceptably assured relative to a concern (such as quality, safety, security or privacy) in the intended operating environment.
The operating environment includes:
1. Policies related to the use of data in the organization (privacy policy, regulation policy, ...).
2. Data quality policies defined by the organization.
3. Risk to be mitigated in the use, consumption and sharing of data by the organization.
4. Control directives to be followed in the use, consumption and sharing of data by the organization.

  MetaData Asset
A  MetaData Asset is a Physical Data Asset used for the physical description of data consumed and produced by Software Systems.
 MetaData Assets range from NoSQL Data Schemas to MetaData Fields.
They are defined in MetaData Dictionary(ies) and managed in Data Catalogs.
  MetaData Dictionary   MetaData Dictionary
  MetaData Field
A  MetaData Field is a Physical Data Property that is the specification of the content of a Meta Dataset.
 MetaData Fields can be "reused" by multiple Meta Datasets where they play the role of Identification MetaField, Relationship MetaField or Value MetaField.
A  MetaData Field can be :
1) a Primitive Type that defines the data type of a field, such as "string" or "integer".
2) a Simple Field that defines a simple value such as "customer name" or "delivery date". Simple Fields are often described by their name, primitive type and length.
3) a Structured Field that defines complex values such as "address".
   Meta Field
  MetaData Object
A  MetaData Object is Physical Data Object that represents an element of a physical schema as it will be stored in relational or non relational DBMS (NoSQL, triple store, document store, flat files, etc.).
A  MetaData Object is either a Meta Dataset or a MetaData Field.
