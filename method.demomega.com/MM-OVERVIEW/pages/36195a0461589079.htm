CONCEPT DOMAIN - Architecture GRID Pattern

DescriptionPattern Domain
The  Architecture GRID Pattern domain bundles sub-domains used to model how a system, including an enterprise, operates/functions in its environment.
It includes Agents, Processes, interactions, Behavioral Rules, Capabilityies, Outcomes, Information Objects and Directives which constitute the columns of the EA GRID.

Operating Semantic Pattern Domain
The  Operating Semantic domain is used to model how a system (Agent) operates/functions (Behavior) in its Operating Eco-System to produce Outcomes that benefit (value) to other Agents or affect (have an effect upon) these Agents.
It includes Agents, Processes, Interaction Scenarios, Policy(ies), Capability(ies), Outcomes.
