CONCEPT DOMAIN - Operating Semantic

DescriptionPattern Domain
The  Operating Semantic domain is used to model how a system (Agent) operates/functions (Behavior) in its Operating Eco-System to produce Outcomes that benefit (value) to other Agents or affect (have an effect upon) these Agents.
It includes Agents, Processes, Interaction Scenarios, Policy(ies), Capability(ies), Outcomes.

  Functional Asset
 Functional Assets encompasse Architecture Blocks used to describe how an enterprise function. This includes the purposes of an enterprise (Capability) and the way the enterprise operates to fulfill these purposes (Agents and their  Behaviors).
 Functional Assets include:
1. Blocks defining results of Behaviors of the enterprise or its sub-systems, that benefit to it internal or external customers : Outcome,
2. Blocks used to describe the capabilities of the enterprise or its sub-systems: Capability,
3. Blocks used to describe information: Information Object.
4. Blocks used to describe how the enterprise operates: Operating Assets (Agent, Process, Interaction Scenario, Service Interface).
All  Functional Assets are constrained by Policys and are exposed to Risk Types.

  Information Object
An  Information Object  represents anything that can be communicated or memorized by an Agent in order to produce its Outcomes.
An  Information Object is either an Information Entity or a Information Property.
The difference lies in their relationship to change and to time. Information Entity(ies) can change over time and have a lifeycle while Information Propertys are immutable characteristics.

  Operating Asset
An  Operating Asset is a Functional Asset used to describes the way Outcomes are produced and consumed: how  (Behaviors) and by whom (Agents).
 Operating Assets include  Agents and their Behaviors (Processes, Service Interfaces and Interaction Scenarios).
 Operating Assets fulfill Capability(ies) (Fulfilled Capability).
As any Functional Asset,  Operating Assets are subject to Policy Conformances. In addition, they define Rule Enforcements to indicate how Policy Conformances are met.
Any  Operating Asset can be considered as a Value Added Object which means it can be transformed and delivered as part of business transactions.
   Class of Operating Block
  Operating Context An  Operating Context is a perimeter of Architecture Block (Type) that forms the scope of an Enduring Initiative. It can be a set of Capability(ies) that define the purpose of operating Agents or the Operating Eco-System in which these Agents deliver these Capability(ies).
  Operating Eco-System An  Operating Eco-System is an Operating Context in which an Agent exists or lives for a specific purpose, interacting with partners.
For instance, the operating context of an enterprise includes its customers and suppliers.

  Outcome
An  Outcome is an Event that indicates the availability of an expected change in the state of a Value Added Object brought about by a specific Behavior,
An  Outcome is a connection point between a producer Behavior and a consumer Behavior which benefits from the produced Value Added Object.
In HOPEX, the most common  Outcome is exemplified by the Information Outcome which refers to the production or consumption of a specific Information Object such as an order received or a  payment rejected.

  Policy
A  Policy is a Directive that is not directly enforceable whose purpose is to govern, guide or constrain the structure and behavior of Agents in the enterprise.
Policies provide the basis for rules and govern Behaviors carried out by Agents.
Example of  Policys are Regulation Articles, Business Policys or Architecture principles.

  Process
A  Process is a Behavior that describes a typical course of action intended to produce Outcomes, through the involvement of Agents as Active Participants.
During its course of action, a process consumes or produces Value Added Objects, including Information Objects.
1) It may read or write Information Objects from and to its Process Stores.
2) It may receive Value Added Objects at its boundary:  Consumed Outcomes.
3) It may produce Value Added Objects at its boundary:  Produced Outcomes.
The course of actions of a  Process is constrained (Rule Enforcement) by the application of Behavioral Rules that define what is allowed and not allowed to do.

  Risk Type
A  Risk Type refers to a distinct category or classification of risk based on its origin, nature, or potential impact. It helps in organizing and addressing different sources of uncertainty or potential harm that an individual, organization, or system might face. By categorizing risks into different types, entities can develop more targeted mitigation strategies and response plans.
Common risk types include Operational Risk Type, Privacy Risk Type, and Compliance Risk Type, among others.
   Risk Type
  Service Interface A  Service Interface is a communication Behavior that describes a typical course of interactions, coordinated by Events and Outcomes, and intended to produce Value Added Objects, through the involvement of Agents.

  Value Added Object
A  Value Added Object is an aspect of Architecture Block (Type)s which can be the subject of business transactions.
A  Value Added Object can be created, enhanced or transformed through the action of a producer Agent, resulting in an increase in value (Outcome) that benefits (value) to a user Agent .
 Value Added Objects include all Operating Assets ( Agents,  Behaviors), Information Objects and Natural Resources.
Examples:
- An improved process (Behavior) that is the result of a refactoring activity done by a consulting organization (producer Agent) for the benefit of a business unit (user Agent).
- A new application (Agent) that is the result of development and sales activities of a software company (producer Agent) for the benefit of an end user (user Agent).
