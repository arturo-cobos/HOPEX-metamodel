DEFINITION - Data Area

Description
A  Data Area is subset of the enterprise’s data that are meant to be manipulated together in the context of business operations.
For instance, a "Sales" Data Area contains at least the following entities: Customers, Orders, Products.
Each entity in a  Data Area has CRUD characteristics. For instance, Products in a "Sales" Data Area, while Customers and Orders have all CRUD characteristics.
Data Areas define functional data boundaries used both for Data Allocation to applications (see Data Stores) and Data Governance for data stewardship.
