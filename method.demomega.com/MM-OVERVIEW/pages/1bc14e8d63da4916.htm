DOCUMENTED OBJECTS

NameDescription
  Constraining Property
A  Constraining Property expresses a type of constraint that can be applied to Architecture Blocks.
These constraints can be either Measurable Property (Abstract) (Time, Mass, Cost, ..), Environmental Propertys (Terrain Type, Light conditions, Weather conditions) or a grouping of these (Set of Constraining Properties).
 Constraining Propertys are used to constrain Architecture Blocks in the context of non-functional requirement analysis and Value Proposition analysis.
  Constraint Value
A  Constraint Value is a possible value of a Constraining Property such as a delivery time of 30 minutes, a weight of 20 kg, a high level of confidentiality, stormy conditions.
A  Constraint Value is either a single measure (Qualifying Value (Abstract): a weight of 20 kg), an Environmental Value (stormy conditions.) or a Set of Constraint Values.
 Constraint Values are used to constrain Architecture Blocks in the context of non-functional requirement analysis and Value Proposition analysis.
  Measurable Property (Abstract)
A  Measurable Property (Abstract) is a Constraining Property that expresses a type of measurement (Time, Mass, Cost, ..) and define the units used for measures (minutes, kg, euros, etc.).
Example:
- Time to deliver in minutes.
- Delivery cost in kilo Euros.
 Measurable Property (Abstract)s are described by Measure Unit: kg, Liter, Gallon, Hour, Minute.
HOPEX provides multiple concrete implementation of  Measurable Property (Abstract) such as Measurable Property or Flow Measure Type.
 Measurable Property (Abstract)s are used to constrain Architecture Blocks in the context of non-functional requirement analysis and Value Proposition analysis.
