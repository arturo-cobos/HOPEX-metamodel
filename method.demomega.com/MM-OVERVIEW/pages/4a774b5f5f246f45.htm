  Table
A Table is a relational data structure, used as the reference for the switch to production. The  Table is the central element of the relational databases.
A table is accessible by means of a primary key, and if necessary foreign keys; it is described by an ordered sequence of Columns. A  Table is generally derived from a Logical Data Entity or asssociation.

  Information Dictionary
An  Information Dictionary is a dictionary of  Information Assets used to describe an information architecture. This includes:
1. Information Domain
2. Information Entity
3. Concept Property
Concrete implementations are Business Dictionary, Logical Data Dictionary, etc.
   Architecture Information Container
  Physical Data Dictionary A  Physical Data Dictionary is Data Dictionary that holds the definition of Physical Data Assets such as Physical Data Domain definitions, Table definitions, Column Type definitions.
  Physical Data Map A  Physical Data Map is a top level grouping of Physical Data Domains.   Physical Data Map
  Relational Data Asset
A  Relational Data Asset is a Data Asset used for description of relational data consumed and produced by Software Systems.
 Relational Data Assets range from Relational Schemas to Column Types.
They are defined in Relational Dictionary(ies) and managed in Data Catalogs.
  Relational Entity
A  Relational Entity is a relational data structure that is either a Table or a Table View.
A  Relational Entity is accessible by means of a primary key, and if necessary foreign keys; it is described by an ordered sequence of Columns.
