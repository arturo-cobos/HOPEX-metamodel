  Aggregate Block
An  Aggregate Block is a Building Block which has an internal structure.
This structure is made of Aggregate Members which are either simple Internal Objects or Contextual Objects that project an external Building Block into the in the  Aggregate Block structure.
    References:
  •  DDD - Aggregate
  •  Martin Fowler - Aggregate

  Aggregate Block (Type)
An  Aggregate Block (Type) is a class of  Aggregate Blocks.
It is the Powertype of Aggregate Block (Individual).

  Aggregate Entity
An  Aggregate Entity is a Nesting Entity which has an internal structure made of Aggregate Entity Members.
1. Internal characteristics called Internal Entitys.
2. Contextualization of external Entitys called Contextual Entitys.
    References:
  •  Martin Fowler - Aggregate
  •  OMG - UML - StructuredClassifier
  Building Block
A  Building Block is an autonomous, resusable Entity that contributes to the makeup of other  Building Blocks.
Building Blocks have an independent existence. Because of their independent existence, they must be owned by an independent artifact which sole purpose is model management: Containers.
Examples:
- Functionality(ies), Business Process(es), Data Entity(ies), Tags, ARE  Building Blocks.
- Operations of a process or attributes of a data entity ARE NOT building blocks. They are elements on an internal structure (see Aggregate Block).

  Container
A  Container is a collection of Building Blocks whose existence and meaning are defined by the  Container.
A  Container is both:
1. A lexical scope: the meaning of a Building Block can only be determined by the  Container where it was created.
2. A universe of discourse: the complete range of Building Blocks that are expressed, assumed, or implied in an architecture domain.
Examples:
- Container Package
- Enterprise
    References:

  Encapsulated Aggregate
An  Encapsulated Aggregate is a Aggregate Block where its internal structure and state changes are hidden from the outside world by means of a formal boundary.
The only way to access to an  Encapsulated Aggregate is through this formal boundary, which ensures the aggregate's internal consistency.
Within an  Encapsulated Aggregate structure, Contextual Objects can be linked together through  Block Connections.
Examples:
- Processes, their sequenced operations and their boundary events.
- Applications, their connected IT-services and their APIs boundaries (Service Points and Request Points).
    References:
  •  SysML2 - Structure Concept
  Encapsulated Aggregate (Individual) An  Encapsulated Aggregate (Individual) is a Aggregate Block (Individual) which has an internal structure and a boundary.
