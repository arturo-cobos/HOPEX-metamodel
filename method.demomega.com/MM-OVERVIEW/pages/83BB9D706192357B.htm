Abstract Concepts
ConceptDescriptionHOPEX Metaclass
  Appraisal
An  Appraisal is a time bound statement expressed by stakeholders to qualify Architecture Blocks with respect to their worth.
Assessments and Decisions are two major types of  Appraisals provided by HOPEX.
   Team Appraisal
  Appraisal Option
An  Appraisal Option is a possible value of an Appraisal Type that will be selected during an appraisal activity.
For instance, when making decisions about invesment on an enterprise asset, the possible options can be: eliminate, invest, migrate.
   Appraisal Option
  Appraisal Type
An  Appraisal Type refers to a kind of Appraisal method and technique used to evaluate the worth, effectiveness, compliance, or quality of enterprise Functional Assets.
 Appraisal Types define a set of Appraisal Options which are the possible values that will result from an Appraisal activity.
For instance, when making decisions about invesment on an enterprise asset, the possible options can be: eliminate, invest, migrate.
Assessment Type and Resolution Type are the two appraisal techniques provided by HOPEX.
   Appraisal Type
  Assessment
An  Assessment is the result of the act of judging or assessing an aspect of the enterprise, based on an enterprise description and on the comparison of this description with the actual state of the enterprise.
An  Assessment is made by one or several Stakeholder s.
   Assessment Node
  Functional Asset
 Functional Assets encompasse Architecture Blocks used to describe how an enterprise function. This includes the purposes of an enterprise (Capability) and the way the enterprise operates to fulfill these purposes (Agents and their  Behaviors).
 Functional Assets include:
1. Blocks defining results of Behaviors of the enterprise or its sub-systems, that benefit to it internal or external customers : Outcome,
2. Blocks used to describe the capabilities of the enterprise or its sub-systems: Capability,
3. Blocks used to describe information: Information Object.
4. Blocks used to describe how the enterprise operates: Operating Assets (Agent, Process, Interaction Scenario, Service Interface).
All  Functional Assets are constrained by Policys and are exposed to Risk Types.
