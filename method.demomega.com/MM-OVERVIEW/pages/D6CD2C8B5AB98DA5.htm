  Data Category
A  Data Category is a classification or division of Information Objects regarded as having particular shared characteristics.
Examples of  Data Categorys are Health data, Financial data, Contact data.

ConceptDescriptionHOPEX Metaclass
  Architecture Namespace
An  Architecture Namespace is a Model Namespace  that packages Architecture Blocks used to describe the architecture of the enterprise.
This includes  Information Asset, Processes, Software Systems, Hardware Systems, Organizations, etc.
 Architecture Namespaces are used to split HOPEX repository content into several independent modules (aka package).
They allow virtual partitions of the repository. In particular, Architecture Blocks owned by different  Architecture Namespace(ies) can have the same name (namespacing).
There are two main kinds of  Architecture Namespace: Enterprise and Library.
   Architecture Building Block Container
  Information Asset
An  Information Asset is a Model Building Block involved in the description of information.
 Information Assets range from Information Objects,  Information Domains to terminologies.
They are defined in Information Dictionary(ies) and managed in Data Catalogs.
  Information Dictionary
An  Information Dictionary is a dictionary of  Information Assets used to describe an information architecture. This includes:
1. Information Domain
2. Information Entity
3. Concept Property
Concrete implementations are Business Dictionary, Logical Data Dictionary, etc.

An  Information Entity is an Information Object that is not fundamentally defined by its attributes, but rather by its continuity.
An  Information Entity evolves over time and has states.

  Information Map
An  Information Map is a top level grouping of Information Domains.
  Information Maps are used to provide navigationtop level entry points for Information Dictionary(ies) and to scope data management initiatives at the level of portfolio management ( Data Catalog), at the project level or at the Enterprise level ( Enterprise Concept Map).
   Information Map
  Information Object
An  Information Object  represents anything that can be communicated or memorized by an Agent in order to produce its Outcomes.
An  Information Object is either an Information Entity or a Information Property.
The difference lies in their relationship to change and to time. Information Entity(ies) can change over time and have a lifeycle while Information Propertys are immutable characteristics.
   Information Entity
  Information Property
An  Information Property is an Information Object that represents a characteristic of an Information Entity.
An  Information Property is fundamentally defined by its value. It doesn't evolve over time and is thereby immutable.
For instance "address" or "customer name" are usually considered as  Information Propertys.
