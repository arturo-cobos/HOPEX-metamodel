  Business Document
A Business Document is a document whose lifecycle can be synched to the MEGA repository content: it may be attached to mega objects and a document version bound to a mega object timestamp in the Time Machine. This document can be of any given file format.
The Business Document versions are managed within the MEGA repository and the document storage file in the file system can be cyphered.
   Business Document
  Tag
A  Tag is a general classification description which permits characterization of Entitys.
All Entitys can be tagged.

  Model Building Block
A  Model Building Block is a Building Block used to describe and manage the enterprise. It comprises:
- Architecture assets (Architecture Blocks) used to describe the enterprise.
-  Initiatives used to manage enterprise transformations and assurance objectives and activities.
-  Governance Blocks used to ensure that these objectives and activities are understood and fulfilled.
   Model Building Block
  Model Dictionary
A  Model Dictionary is a Model Namespace which sole purpose is model management.
Examples: Library, Information Dictionary.
Conterexamples: Enterprises or Infrastructure Landscapes are Model Namespaces but are not  Model Dictionaryies because they have functional purposes that go beyond model management.
    References:
  •  OMG - KerML - LibraryPackage
  •  OMG - UML - Package
  Model Namespace
A  Model Namespace is a namespace of Model Building Blocks used to describe both the architecture of the enterprise and Initiatives aimed at designing, developing, running, adapting and maintaining this architecture over time.
 Model Namespaces range from Information Dictionary(ies), architecture Library(ies) up to an entire Enterprise .
