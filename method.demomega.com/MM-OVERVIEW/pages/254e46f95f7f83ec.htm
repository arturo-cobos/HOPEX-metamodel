BUILDING BLOCK - Meta Dataset

Description
A  Meta Dataset is a Physical Data Entity that represents the structure of a collection of related data held in a data source. The represented data structure can be relational (Table) or NoSQL (triple store, document store, flat files, etc.).
A  Meta Dataset consists of three types of MetaData Field:
1) An Identification MetaField, sometimes called "primary key".
2) Relationship MetaFields used to represent relationships with other  Meta Datasets.
3) Value MetaFields used to store values in the  Meta Dataset.
 Meta Datasets can be defined at design time in the context of data architecture activities. They can also be imported automatically through data discovery.
