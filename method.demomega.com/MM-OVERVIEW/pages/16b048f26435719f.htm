DEFINITION CLASSIFIED CONCEPTS

CATEGORY - EA Block Management

Description
Parent Category  EA Ontology   
Classified Concepts
ConceptDescription
  Annotation Block An  Annotation Block is a Building Block used to provided additional characteristics such as sketching, reports, description notes.
    References:
  •  WordNet - Annotation
  Architecture Block An  Architecture Block is any Model Building Block used to describe an enterprise architecture. This includes types (applications, org-units, ...) and individuals (physical persons, software installations, sites, ...).
    References:
  •  OMG - UAF - UAFElement
  •  OpenGroup - TOGAF - Architecture Building Blocks
  Architecture Block (Individual)
An Individual which is used in the description of the enterprise.
 Architecture Block (Individual) can be classified by Constraint Value
    References:
  •  OpenGroup - TOGAF - Architecture Building Blocks
  Architecture Block (Type) An  Architecture Block (Type) is a type of Architecture Block which is used in the description of the enterprise, such as Processes, Org-Units, Applications.
    References:
  •  OpenGroup - TOGAF - Architecture Building Blocks
  Architecture Category An  Architecture Category is a Categorization Block used to classify Architecture Blocks. Examples are Data Category, Business Line.
  Architecture Namespace
An  Architecture Namespace is a Model Namespace  that packages Architecture Blocks used to describe the architecture of the enterprise.
This includes  Information Asset, Processes, Software Systems, Hardware Systems, Organizations, etc.
 Architecture Namespaces are used to split HOPEX repository content into several independent modules (aka package).
They allow virtual partitions of the repository. In particular, Architecture Blocks owned by different  Architecture Namespace(ies) can have the same name (namespacing).
There are two main kinds of  Architecture Namespace: Enterprise and Library.
  Business Document
A Business Document is a document whose lifecycle can be synched to the MEGA repository content: it may be attached to mega objects and a document version bound to a mega object timestamp in the Time Machine. This document can be of any given file format.
The Business Document versions are managed within the MEGA repository and the document storage file in the file system can be cyphered.
  Categorization Block A  Categorization Block is a Model Building Block used to provid classification and qualification of other Model Building Blocks.
For instance, Tags are used for classification. Constraint Value are used for qualification or quantification.
  Class of Model Block A  Class of Model Block is a Class that classifies Model Building Blocks.
  Constraining Property
A  Constraining Property expresses a type of constraint that can be applied to Architecture Blocks.
These constraints can be either Measurable Property (Abstract) (Time, Mass, Cost, ..), Environmental Propertys (Terrain Type, Light conditions, Weather conditions) or a grouping of these (Set of Constraining Properties).
 Constraining Propertys are used to constrain Architecture Blocks in the context of non-functional requirement analysis and Value Proposition analysis.
    References:
  •  OpenGroup - TOGAF - Content Metamodel - Measure
  •  Russell Ackoff - Choice & Communication - MorphologicalProperty
  •  Wikipeddia - List of quality attributes
  •  Wikipedia - Measurement
  Constraining Property Type
A  Constraining Property Type is a class of  Constraining Property.
Example: measurement types (Performance, Security, Cost), PESTEL (political, economic, social, environmental, legal)
It is used to group Constraining Property(ies) by domain of concerns represented by Schema of Measurement.
    References:
  •  ISO 15926 - ClassOfProperty
  •  Russell Ackoff - Choice & Communication - FunctionalClass
  Constraint Value
A  Constraint Value is a possible value of a Constraining Property such as a delivery time of 30 minutes, a weight of 20 kg, a high level of confidentiality, stormy conditions.
A  Constraint Value is either a single measure (Qualifying Value (Abstract): a weight of 20 kg), an Environmental Value (stormy conditions.) or a Set of Constraint Values.
 Constraint Values are used to constrain Architecture Blocks in the context of non-functional requirement analysis and Value Proposition analysis.
    References:
  •  Britannica - Measurement scale
  •  ISO 15926 - Property
  •  Wikipeddia - List of quality attributes
  •  Wikipedia - Level of measurement
  Environmental Constraint Type
An  Environmental Constraint Type is a class of  Environmental Property.
Example: PESTEL (political, economic, social, environmental, legal),  physical conditions (weather conditions, terrain conditions,)
It is used to group Environmental Property(ies) by domain of concerns represented by Schema of Measurement.
  Environmental Property
An  Environmental Property is a characteristic of the environment in which an Enterprise or its sub-systems are required to operate.
Examples are Terrain Type (e.g. muddy, frozen ground, deep snow, etc.), light conditions (e.g. broad daylight, dusk, moonlit, etc.), weather condition, polical conditions (political stability, level of education), economical conditions (exchange rates, inflation rate).
  Environmental Value
An  Environmental Value is a specific value of an Environmental Property.
 Environmental Values are used as constraints on Architecture Blocks in the context of non-functional requirement analysis and Value Proposition analysis.
Examples:
- Muddy, frozen ground, deep snow (Terrain Type)
- Broad daylight, dusk, moonlit  (Light condition)

  Governance Block  Governance Blocks include all instruments used to monitor and steer transformation Initiatives. They include incidents, assessments, Workflows, indicators and their templates.
  Governance Namespace Container of Building Blocks used to measure, assess, monitor transformation  Initiatives .
  Initiative
An  Initiative is any piece of work that is undertaken or attempted under the responsibility of a Steering Authority.
 Initiatives range from single project to long term initiatives (Enduring Initiative) such as Enterprises.
 Initiatives are sychronized over time using Initiative Milestones.
    References:
  •  Merriam Webster - Undertaking
  •  Wordnet - Undertaking
  Initiative Block An  Initiative Block is a Model Building Block used to describe enterprise transformation.
  Library
A  Library is a kind of Architecture Namespace used to split HOPEX repository content into several independent modules (aka package).
They allow virtual partitions of the repository. In particular, Architecture Blocks owned by different  Library(ies) can have the same name (namespacing).
  Measurable Property
A  Measurable Property is a concrete Measurable Property (Abstract) that can be applied to all Architecture Block (Type)s.
For instance, "Cost" can be applied to all Architecture Blocks, while "weight" likely applies only to Hardware Systems.
  Measurable Property (Abstract)
A  Measurable Property (Abstract) is a Constraining Property that expresses a type of measurement (Time, Mass, Cost, ..) and define the units used for measures (minutes, kg, euros, etc.).
Example:
- Time to deliver in minutes.
- Delivery cost in kilo Euros.
 Measurable Property (Abstract)s are described by Measure Unit: kg, Liter, Gallon, Hour, Minute.
HOPEX provides multiple concrete implementation of  Measurable Property (Abstract) such as Measurable Property or Flow Measure Type.
 Measurable Property (Abstract)s are used to constrain Architecture Blocks in the context of non-functional requirement analysis and Value Proposition analysis.
    References:
  •  ISO 15926 - ClassOfProperty
  •  Wikipeddia - List of quality attributes
  Measurement Type
A  Measurement Type is a class of  measurement.
Example: Performance, Security, Cost.
It is used to group Measurable Property (Abstract)(ies) by domain of concerns represented by Schema of Measurement.
  Model Building Block
A  Model Building Block is a Building Block used to describe and manage the enterprise. It comprises:
- Architecture assets (Architecture Blocks) used to describe the enterprise.
-  Initiatives used to manage enterprise transformations and assurance objectives and activities.
-  Governance Blocks used to ensure that these objectives and activities are understood and fulfilled.
  Model Dictionary
A  Model Dictionary is a Model Namespace which sole purpose is model management.
Examples: Library, Information Dictionary.
Conterexamples: Enterprises or Infrastructure Landscapes are Model Namespaces but are not  Model Dictionaryies because they have functional purposes that go beyond model management.
    References:
  •  OMG - KerML - LibraryPackage
  •  OMG - UML - Package
  Model Namespace
A  Model Namespace is a namespace of Model Building Blocks used to describe both the architecture of the enterprise and Initiatives aimed at designing, developing, running, adapting and maintaining this architecture over time.
 Model Namespaces range from Information Dictionary(ies), architecture Library(ies) up to an entire Enterprise .
    References:
  •  OMG - UML - Namespace
  Qualifying Value
A  Qualifying Value is a Qualifying Value (Abstract) that can be applied to all Architecture Blocks.
For instance, "Cost" is a  Qualifying Value.
  Qualifying Value (Abstract)
A  Qualifying Value (Abstract) is a possible value of a Measurable Property (Abstract) such as a delivery time of 30 minutes, a weight of 20 kg, a high level of confidentiality.
It represents how much there is or how many there are of something that you can quantify according to its Measurable Property (Abstract).
 Qualifying Value (Abstract)s are used as constraints on Architecture Blocks in the context of non-functional requirement analysis and Value Proposition analysis.
    References:
  •  ISO 15926 - Property
  Reference Dictionary
A  Reference Dictionary is an aspect of Model Namespace that are used as reference libraries used to hold reference materials used to develop architectures.
Example:
- Vendor Catalog
- Regulatory Framework
-
    References:
  •  OpenGroup - TOGAF - Method - Reference Libraries
  Schema of Category
A  Schema of Category is a Top level group of categories that belong to a common domain.
For instance, the GDPR categorization schema groups all categories relevant to GDPR such as data categories, sensitive activities.
  Schema of Measurement A  Schema of Measurement is a top level grouping of Constraining Property(ies) that belong to a common domain.
  Set of Constraining Properties
A  Set of Constraining Properties is a combination of Constraining Propertys.
For example, "delivery time" at "appropriate cost" in "stormy conditions" is a  Set of Constraining Properties.
    References:
  •  OMG - UAF - MeasurementSet
  Set of Constraint Values
A  Set of Constraint Values is a combination of Constraint Values.
Example: delivery time of 30 minutes, at a cost ranging between $8.50 and $10, in stormy conditions.
    References:
  •  OMG - UAF - ActualMeasurementSet
  Sketch A  Sketch is a rough draft of a model; unformal representation.
    References:
  •  OMG - UAF - HighLevelOperationalConcept
  Tag
A  Tag is a general classification description which permits characterization of Entitys.
All Entitys can be tagged.
