DEFINITION CLASSIFIED CONCEPTS

CATEGORY - Resource Ontology

DescriptionThe  Resource Ontology classifies abstract concepts describing share characteristics of all of type resources.
Parent Category  Resource Level   
Classified Concepts
ConceptDescription
  Business Rule
An  Business Rule is a rule that is under business jurisdiction.
A rule’s being under "business jurisdiction" means that it is under the jurisdiction of the community that it governs or guides - that the community can opt to change or discard the rule. Laws of physics may be relevant to an Enterprise; legislation and regulations may be imposed on it; external standards. These things are not  Business Rules from the company’s perspective, since it does not have the authority to change them. The company will decide how to react to laws and regulations, and will create  Business Rules to ensure compliance with them. Similarly, it will create  Business Rules to ensure that standards or best practices are implemented as intended.
  Business Scenario
A  Business Scenario is a story describing how Business-Entity (Org-Unit) interact wtiht its partners (Partner Org-Unit) to achieve Business Outcomes in a specific Eco-System (Business Environment).
This includes:
1) A course of events represented by  Business Object Flows depicting the story towards the delivery of expected Business Outcomes.
2) Interacting  Business Entities (Org-Units)  who participate to the story in the considered Eco-System.
  Business System Asset A  Business System Asset is man made resource that is a means to achieve Business Outcomes.
  Business-Entity Ecosystem
An  Business-Entity Ecosystem is a Resource Ecosystem used to describe the operating context in which a Business-Entity (Org-Unit) exists or lives for a specific purpose.
For instance, the operating context of a company includes its customers and suppliers.
  Business-Entity Scenario
A  Business-Entity Scenario is a story describing how the components of a Business-Entity (Org-Units) interact to produce Business Outcomes.
This includes:
- A course of events represented by information flows depicting the steps towards the delivery of expected Outcomes.
- Components of an Org-Unit (Sub-Org-Units or Organizational Position) that participate to the story.
  Data Asset
A  Data Asset is an Information Asset used for the description of data consumed and produced by Software Systems.
They are defined in Data Dictionary(ies) and managed in Data Catalogs.
    References:
  •  NIST - Data Asset
  Data Dictionary
A  Data Dictionary is a dictionary of Data Assets used to describe a data architecture. This includes:
1. Data Domains,
2. Data Entity(ies),
3. Data Property(ies).
Concrete implementations are Logical Data Dictionary(ies), Relational Dictionary(ies), MetaData Dictionary(ies).
    References:
  •  OMG - UML - Package
  Data Domain
A  Data Domain is subset of the enterprise’s data that are meant to be manipulated together in the context of business operations.
For instance, a "Sales"  Data Domain contains at least the following entities: Customers, Orders, Products.
Each Data Entity in a  Data Domain has CRUD characteristics. For instance, Products are read-only in a "Sales"  Data Domain, while Customers and Orders have all CRUD characteristics.
 Data Domains define functional data boundaries used both for Data Allocation to Business Systems (see Data Stores) and Data Governance for data stewardship (Data Catalogs).
  Data Entity A  Data Entity is an abstract structure of an Information Entity. It has an independent existence and can be uniquely identified.
Data Entity is characterized by Relationships it has with other  Data Entity(ies) and by Value Property(ies).
    References:
  •  DDD - Glossary - Entity
  •  OpenGroup - TOGAF - Definition - Data Element
  Data Map A  Data Map is an urbanization tool for  Data Asset. It represents a set of Data Domains in a particular context.
  Data Object
A  Data Object  represents the abstract structure of any kind of data that can be memorized by a Software System.
A  Data Object is either a Data Entity or a Data Property.
Only Data Entitys can have identity and states. Data Propertys only handle raw data.
    References:
  •  NIST - Data Asset
  •  UCF Glossary - Data
  •  UCF Glossary - Data Element
  Data Property
A  Data Property is a Data Object that is fundamentally defined by its value. It doesn't evolve over time is thereby immutable.
It represents a logical view of a data structure that stored in data stores.
For instance an address or a customer name are usually considered as  Data Property(ies).
    References:
  •  DDD - Glossary - Value Object
  •  UCF Glossary - Data Field
  Natural Resource A  Natural Resource is a kind of Resource Operating Asset that occurs in nature such as oil, water, gas or coal.
    References:
  •  OMG - UAF - NaturalResource
  Resource Agent
A  Resource Agent is a Business Operating Agent which represents physical resources that produce Business Outcomes of the enterprise.
A  Resource Agent can be a Human Resource (Organizational Position or Org-Unit), a Business System (Software System or Hardware System) or a Capability Configuration (an assembly of Human Resources and Business Systems.)
    References:
  •  OMG - UAF - ResourcePerformer
  •  OpenGroup - TOGAF - Definition - Business System
  Resource Agent Environment A  Resource Agent Environment is an operating context which defines the interactions ( Business Service Channel) of a Resource Agent with its partners (Partner Resource Agent).
  Resource Behavior A  Resource Behavior is a Resource Operating Asset that describes any action or reaction of a Resource Agent to external or internal circumstances. This includes Resource Processes (actions), Resource Interaction Scenarios (stories) or service interactions (Business Service Interface).
  Resource Configuration
A Resource Configuration is a generic System of Resources that represensed System of Resources that are not Facilitys.(Data centers, Factories, Outlets).
Examples:
- Configured Nuclear power plant (with operating teams), 
- Configured aircraft carrier (with crew), Configured rescue ship (with rescue team).
When specific characteristics need to be considered for a given type of  Resource Configuration, it is recommended to create a new sub-type of System of Resources to host these characteristics.
  Resource Ecosystem A  Resource Ecosystem is an Operating Eco-System  that describes the operating context in which a Resource Agent exists or lives for a specific purpose.
  Resource Environment Scenario
A  Resource Environment Scenario is a story that describes how a Resource Agent interacts with its partners (Partner Resource Agents) to achieve Business Outcomes in a specific Eco-System (Resource Agent Environment).
This includes:
1) A course of events represented by  Business Object Flows depicting the story towards the delivery of expected Business Outcomes.
2) Interacting  Resource Agents who participate to the story in the considered Eco-System.
  Resource Operating Asset  Resource Operating Assets comprise physical assets which contribute to the delivery of Business Outcomes of the enterprise. This includes Resource Agents, their behaviors (Resource Behavior:  Resource Process, Resource Interaction Scenario), and Natural Resources.
    References:
  •  OMG - UAF - CapableElement
  •  OpenGroup - TOGAF - Definition - Solution Building Block
  •  OpenGroup - TOGAF - Guide - Solution Building Blocks
  Resource Process
A  Resource Process is a Business Operating Process which is conducted by Resource Agents (physical resources such as Human Resource, System of Resources ,Business Systems) and which describes how these entities act to deliver Business Outcomes.
 Resource Processes include Business Processes and System Processes.
  Resource Rule A  Resource Rule is an Business Operating Rule that is enforced by Resource Operating Assets of the Enterprise.
    References:
  •  OMG - BMM - Business Rule
