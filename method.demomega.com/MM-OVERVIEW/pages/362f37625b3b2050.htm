  Meta Dataset
A  Meta Dataset is a Physical Data Entity that represents the structure of a collection of related data held in a data source. The represented data structure can be relational (Table) or NoSQL (triple store, document store, flat files, etc.).
A  Meta Dataset consists of three types of MetaData Field:
1) An Identification MetaField, sometimes called "primary key".
2) Relationship MetaFields used to represent relationships with other  Meta Datasets.
3) Value MetaFields used to store values in the  Meta Dataset.
 Meta Datasets can be defined at design time in the context of data architecture activities. They can also be imported automatically through data discovery.

  Simple Field
A  Simple Field is a MetaData Field that represent a single piece of data. A  Simple Field cannot be decomposed.
 Simple Fields are often described by their name, primitive type and length.
For instance, "customer name", "customer id" and "stree number" are usually considered as  Simple Fields.

  Stored Procedure
A  Stored Procedure is a is a subroutine available to applications that access a relational database management system (RDBMS).
It is a kind of  Resource Process handled by RDBMS.
   Stored Procedure
  Structured Field
A  Structured Field is a MetaData Field which is a combination of other MetaData Fields.
For instance, an address in composed of a street field, a city field and a country field.
   Structured Meta Field
  Table
A Table is a relational data structure, used as the reference for the switch to production. The  Table is the central element of the relational databases.
A table is accessible by means of a primary key, and if necessary foreign keys; it is described by an ordered sequence of Columns. A  Table is generally derived from a Logical Data Entity or asssociation.

  Information Dictionary
An  Information Dictionary is a dictionary of  Information Assets used to describe an information architecture. This includes:
1. Information Domain
2. Information Entity
3. Concept Property
Concrete implementations are Business Dictionary, Logical Data Dictionary, etc.
   Architecture Information Container
  MetaData Asset
A  MetaData Asset is a Physical Data Asset used for the physical description of data consumed and produced by Software Systems.
 MetaData Assets range from NoSQL Data Schemas to MetaData Fields.
They are defined in MetaData Dictionary(ies) and managed in Data Catalogs.
  MetaData Dictionary   MetaData Dictionary
  MetaData Field
A  MetaData Field is a Physical Data Property that is the specification of the content of a Meta Dataset.
 MetaData Fields can be "reused" by multiple Meta Datasets where they play the role of Identification MetaField, Relationship MetaField or Value MetaField.
A  MetaData Field can be :
1) a Primitive Type that defines the data type of a field, such as "string" or "integer".
2) a Simple Field that defines a simple value such as "customer name" or "delivery date". Simple Fields are often described by their name, primitive type and length.
3) a Structured Field that defines complex values such as "address".
   Meta Field
  MetaData Object
A  MetaData Object is Physical Data Object that represents an element of a physical schema as it will be stored in relational or non relational DBMS (NoSQL, triple store, document store, flat files, etc.).
A  MetaData Object is either a Meta Dataset or a MetaData Field.
   MetaData Object
  Physical Data Dictionary A  Physical Data Dictionary is Data Dictionary that holds the definition of Physical Data Assets such as Physical Data Domain definitions, Table definitions, Column Type definitions.
  Physical Data Map A  Physical Data Map is a top level grouping of Physical Data Domains.   Physical Data Map
  Relational Data Asset
A  Relational Data Asset is a Data Asset used for description of relational data consumed and produced by Software Systems.
 Relational Data Assets range from Relational Schemas to Column Types.
They are defined in Relational Dictionary(ies) and managed in Data Catalogs.
  Relational Entity
A  Relational Entity is a relational data structure that is either a Table or a Table View.
A  Relational Entity is accessible by means of a primary key, and if necessary foreign keys; it is described by an ordered sequence of Columns.
